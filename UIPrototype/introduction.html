通过课程的学习，能够了解何为界面，何为用户界面，用户界面发展的过程，界面设计方法，GUI小部件，常用的界面原型设计工具以及界面设计准则。 课程主要内容如下： 课时1：何为界面界面 界面，外文名为Interface，又名接口。接口可分为三类，即硬件接口、软件接口和人机接口。人机接口即人与计算机的接口，包括键盘、鼠标、显示器和音箱等。在人机接口中，用户界面通常指在显示器上呈现的软件的界面，包括Batch Interface(1945-1968)、TUI(Text-based User Interface）、CLIS（Command-line user Interface，1969-今）、GUI(Graphic User Interface，1968-今）、WUI(Web-based User Interface）和TUI(Touch User Interface），对于产品设计工作者来说，大部分工作集中在GUI、WUI、TUI（Touch User Interface）的界面设计上。 课时2：界面设计方法 课程讲述了界面设计的三个步骤和GUI小部件的分类。界面设计的三个步骤分别为：交互设计、接口设计和原型设计。交互设计包括以用户为中心的设计（UCD）、用户角色扮演（persona）、面向行为的设计（activity-oriented design）、面向场景的设计（Scenario-based design）和弹性设计（resiliency design）；接口设计包括use case和Constrain enforcement；原型设计则基于交互设计，需要使用控件库。原型设计本身有两个作用：一是与用户交流，对用户的业务进行了解，二是当把原型修改到接近、达到用户需求后，通过原型和文档描述与内部的开发团队交流并达成共识，明确未来所做出的产品的样子。在界面设计中，部件可按照命令选择、集合数据显示、容器、导航、文本输入和输出进行分类，还可以从输入、操作和输出三方面分为三大类。 课时3：GUI小部件 本课程提供了可以学习用在windows客户端和网页上的GUI小部件的两个链接资料（ https://en.wikipedia.org/wiki/Widget(GUI) 和 https://msdn.microsoft.com/library/windows/desktop/dn742399.aspx ）并以Balloons、Check boxes等为例讲述了学习方法。另外需要注意，不同的UI类型，部件之间也会有差异。 课时4：界面原型设计工具 界面原型设计工具可分为客户端工具和在线工具（在线工具包括Processon和墨刀）。客户端Axure RP软件需要安装且收费，它做出的界面原型会生成一个打包文件，解压后将文件夹打开的若干个页面，可供用户用浏览器在本地浏览。此外，打包生成的网页之间有客户端脚本，页面之间有跳转动作；墨刀是在线做界面原型，也可以让界面之间有跳转；相对于Axure RP和墨刀来说，Processon做出的原型只是静态图，跳转关系只能在图上用箭头来表示。Processon上做出的界面原型可以发布出来，用户可通过获得的地址进行在线访问。三种工具各有特色，可根据个人习惯进行选择使用。 课时5：界面设计准则 课程从清晰、一致、简洁、友好、响应、高效和容错七个方面介绍了界面设计的准则。设计界面时，文字描述和布局要清晰；按照自左而右、自上而下的阅读习惯要在右上角显示最重要的内容以及要考虑到容错，即设计出的产品对用户错误的操作要有友好的提示等等，在设计界面时需要遵循这些准则。
